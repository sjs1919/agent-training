4. 技术架构：ME-WE-MA 三层飞轮

衔远大观的架构不是三个 AI 模块的堆叠，而是一套完整的飞轮系统：ME 是入口，WE 是能力，MA 是引擎。

4.1 ME 数字分身（入口层）

能力说明
长期记忆跨会话、跨时间积累岗位背景、决策偏好、工作习惯
专家调度用户只需说出需求，ME 自动判断该调哪个专家
跨端一致飞书、企微、衔远大观，记忆同步、人设一致
分身进化每次使用都在变得更聪明——越用越懂你

关键区分：ME 不是聊天机器人，也不是静态个人知识库，而是在企业语境中代表人持续推进工作的受治理代理——理解目标、偏好、职责、权限、风险边界和历史上下文，在关键节点请求确认。

4.2 WE 专家网络（能力层）

WE 把战略、研发、供应链、销售、运营以及建筑、金融、电商等行业能力封装为专家网络：

  • 战略管理：复盘会议、跟进商机、监控竞情、生成高管周报
  • 研发创新：30 分钟编制复杂工程计划，延误风险提前暴露
  • 供应链管理：提前 7 天预判需求波动，招采从 3-5 天压缩到数小时
  • 销售商务：商机发现、方案生成、客户洞察贯穿从线索到成单
  • 运营决策：人才、会议、经营洞察进入 CEO 一屏掌控

关键设计：没有目标、权限、证据、接口、语义标准和治理边界，再多 Agent 只是噪声。有了组织上下文，一群 Agent 才能变成真正的硅基组织。

4.3 MA 进化引擎（驱动层）

MA 让任务结果不再散落在个人、文档和群聊里，而是进入组织级进化引擎：

  1. 经验沉淀：评估表现，把高质量路径留下来
  2. 专家评估：通过私有化评测标尺评估专家表现
  3. 自动优化：受控进化反哺 ME 与 WE，飞轮持续加速

4.4 三层进化飞轮

5. L4 能力转化层三大核心技术

5.1 通专融合（General-Specialist Fusion）

创始人周伯文 2022 年率先提出的技术路径：

  • 通用大模型提供语言理解、逻辑推理和跨领域知识
  • 专业化小模型注入行业数据、专家规则和垂直判断
  • 深度融合：两者深度协作，突破任何单一模型的能力天花板

这不是简单的模型微调或 RAG，而是在判断底座层面的融合——通用推理、行业数据和专家规则被压到同一判断框架中。

5.2 数字风洞（Digital Wind Tunnel）

衔远的创新概念——上线前让 Agent 在仿真业务环境中演练：

  • 从行业数据和专家知识构建生成式仿真环境
  • Agent 在交互式业务场景中提前排练
  • 业务执行结果（outcome）转化为 RL 训练的奖励信号
  • 仿真环境 + 训练数据的累积是企业级 AI 部署最稀缺的资产

"训练瓶颈不是算力——是缺乏支持大规模交互的业务环境。"

5.3 Meta-Agent 元智能体

Meta-Agent 是 MA 进化引擎的核心：

  • 识别目标：理解用户意图和业务上下文
  • 拆解任务：将复杂任务分解为可执行步骤
  • 调度专家：动态匹配和调度 WE 专家网络
  • 反哺进化：沉淀如何创造、评估和升级新的专家能力的方法

6. 产业经验飞轮方法论

核心公式：企业的护城河 = 私有化评估 + 可持续进化。模型底座可更换而资产不动，企业上下文图谱全程不出企业边界。

6.1 五步飞轮闭环

  1. 真实执行：智能体在真实企业环境中工作
  2. 私有评测：以企业私有标尺做私有化评测
  3. 经验沉淀：将工作经验沉淀为技能和记忆等资产
  4. 受控进化：新资产通过评测门控实现受控进化
  5. 变强执行：回到执行环节变得更强，飞轮持续加速

6.2 "量控驱装"四步工程路径

步骤含义产出
量建立评测标尺EnterpriseClawBench + NatureBench
控发现进化约束三个反直觉事实（见下）
驱驱动进化引擎Frontis-MA1 元智能体模型
装部署企业系统Frontis Horizon + Leadeep AI

6.3 三个反直觉发现

发现一："模型够强就行"不成立。32 组智能体框架与前沿模型组合实测中，最强组合得分仅 0.663（满分 1.0），平均 0.486。

发现二："框架不重要"不成立。同一模型仅更换执行框架，得分即从 0.62-0.64 跌至 0.458——框架与模型同等重要。

发现三："给 Agent 加记忆就进化"不成立。注入劣质经验后，智能体得分从 0.756 跌至 0.555——经验污染是必然，受控进化是解。

8. 开源贡献与 GitHub 生态

8.1 EnterpriseClawBench（GitHub 开源）

仓库地址：github.com/FrontisAI/EnterpriseClawBench

  • 定位：企业级 Agent 系统评测基准——不是评测模型，而是评测完整 Agent 系统（harness + model）
  • 规模：852 个可复跑的企业真实工作任务
  • 特点：从真实企业 Agent 会话中通过隐私保护管道构建；Agent 需要操作工作区、理解文件、使用工具、交付可用业务产物
  • 开放内容：脱敏公开 raw-session 示例、基准构建管道、本地评测管道、沙箱运行目录协议、参考运行、聚合排行榜数据和可视化
  • 论文：HuggingFace Papers (2606.23654)

8.2 Frontis-MA1（全栈开源）

WAIC 2026 发布的 35B 元智能体模型——数据、代码、框架、权重全部开源。MLE-Bench 与 GPT-5.5 并列第一，参数量仅为其不到百分之一。

8.3 NatureBench

由 5500 篇《Nature》系列论文自动构建出 90 个容器化高价值任务，覆盖 6 大科学领域。最强智能体仅有 17.8% 的任务能超越论文 SOTA。

8.4 同类开源平台对比

平台定位与 Frontis 对比

关键差异：EnterpriseClawBench 是首个从真实企业会话构建的 Agent 评测基准——不评测模型能力，而是评测完整 Agent 系统在真实业务中的表现。这与衔远"私有化评估"理念一致——外部 benchmark 告诉你模型通用能力，private eval 才能告诉你 AI 是否真的在业务中创造价值。

9.2 独特性

10. 护城河、风险与展望

10.1 五道护城河

  1. L4 定位稀缺性：需要通专融合×全栈RL×行业仿真三种能力同时具备，团队门槛极高
  2. EnterpriseClawBench 评测壁垒：852 个企业真实任务的评测体系是基础设施级资产，先发优势明显
  3. 数字风洞仿真环境：仿真环境+训练数据是企业级 AI 部署最稀缺资产，累积即壁垒
  4. 创始人学术+产业双重壁垒：周伯文是 Transformer 核心思想贡献者，学术声誉带来人才吸引力和行业信任
  5. "模型可换、资产不动"的企业锁定：企业积累的 DNA、Memory、Process 构成极高切换成本

10.2 主要风险

  1. 战略转型验证风险：从 ProductGPT（产品创新）到 Frontis Horizon（企业AI原生组织）是根本性转型，商业化路径需要重新验证
  2. L4 定位的教育成本：市场对"能力转化层"概念需要认知教育，销售周期可能较长
  3. 大模型厂商向 L4 扩展：如果 OpenAI/Anthropic/国内大模型厂商自行做通专融合，L4 层可能被压缩
  4. 受控进化的工程复杂度："经验污染是必然，受控进化是解"在理论上成立，但工程落地难度极高
  5. 开源与商业化的张力：Frontis-MA1 全栈开源有利于生态建设，但也可能降低商业壁垒
  6. 供应链场景的宽度：当前标杆案例集中在供应链和建筑，横向扩展到更多行业需要重新积累

10.3 展望

衔远科技赌的是一个更大的命题：不只是"帮企业用 AI"，而是"帮企业拥有自己的智能系统"。如果"产业经验飞轮"方法论成立——模型底座可更换而企业资产不动——那衔远将成为 AI 时代企业智能资产的基础设施提供商。

Frontis-MA1 在 MLE-Bench 上以 1/100 的参数量与 GPT-5.5 并列第一，证明"小模型+Meta-Agent"路线的可行性。这不仅是产品发布，更是对"模型越大越好"这一行业共识的挑战。
