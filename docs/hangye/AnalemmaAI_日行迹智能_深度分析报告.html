公司名由来："Analemma"取自天文学术语"日行迹"——太阳在一年中于天空划出的8字形轨迹。这个名称隐喻了循环、积累与探索的理念，与公司构建"无限心智"的愿景一脉相承。

1.2 战略定位：AI4AI科研自动化基础设施

日行迹智能的战略定位极为独特——在所有AI Agent平台中，它是唯一专注于"AI4AI"（用AI研究AI）范式的公司。其核心使命是：

"将人类进行科学发现的方式，从依赖个人能力的手工作坊，转变为可规模化的工业生产。"

与之前分析的其他Agent平台截然不同：

  • 其他平台解决的是"企业效率"问题（自动化工作流、客服、RPA等）
  • 日行迹解决的是"科学研究本身"的自动化问题——让AI自主完成文献调研、假设生成、实验设计、代码编写、论文撰写的完整科研链路
  • 这不是一个企业级SaaS产品，而是一个研究型实验室的基础设施

1.3 核心理念：研究系统的第一性原理

创始人孙天祥认为，以人为中心的研究系统存在结构性约束：

人类研究的结构性问题FARS的解决方案
门槛高、人才少 AI 7x24小时不间断运行，无人才瓶颈
"反复造轮子" 共享文件系统积累持久记忆，知识可复用
"只重视成功结果，忽略失败结果" 鼓励报告失败结果——"无论验证结果正负向，都构成有意义的知识"
试错成本高 算力换智能——每篇论文成本约$1000，人类需3-6个月

三、产品体系：Lemma + FARS双轨

3.1 Lemma：辅助驾驶模式

在FARS之前，日行迹先上线了Lemma——一个科研辅助工具，定位为"辅助驾驶"模式：

功能说明
快速文献调研AI辅助阅读和整理文献
深度调研报告生成领域综述和研究方向分析
代码实验辅助编写和运行实验代码

孙天祥的定位很清晰："Lemma是辅助驾驶，定位是生产力工具；FARS是自动驾驶，定位是基础设施。"Lemma的核心目的不仅是产品化，更重要的是收集用户反馈以优化FARS系统。

3.2 FARS：自动驾驶模式

FARS（Fully Automated Research System，全自动研究系统）是日行迹的核心产品——一个无人类干预、纯AI驱动的端到端研究系统。它能自主完成：

  1. 文献调研（Literature Review）
  2. 假设生成（Hypothesis Generation）
  3. 代码编写（Code Writing）
  4. 实验执行（Experiment Execution）
  5. 论文撰写（Paper Writing）

这是全球首个以大规模公开直播方式部署的全自动科研系统。

4.3 共享文件系统：Agent间的"记忆总线"

FARS的四智能体之间不直接对话，而是通过共享文件系统交接工作。这个设计选择非常关键：

  • 文件即接口：每个Agent的输出写入文件，下一个Agent从文件读取输入——松耦合、可审计
  • 文件即记忆：文件系统同时承担工作空间和持久记忆功能——跨任务知识可复用
  • 文件即证据：所有中间过程可追溯、可复现——GitHub实时公开提交

4.4 模型策略：多模型混合 + 自研后训练

FARS并非依赖单一模型，而是采用多模型混合策略：

  • 闭源模型API：Claude、GPT、Gemini等多家一线模型按场景调用
  • 自研后训练模型：在部分链路使用经过后训练的自研模型
  • 核心能力在Agent系统工程：不在于训练更强的模型，而在于如何编排多智能体协同完成跨越数天的长链条任务，让系统在无人干预下持续可靠运转

六、论文质量评估与社区反馈

6.1 AI审稿评估

团队使用斯坦福大学开发的Agentic Reviewer（paperreview.ai），按ICLR评审标准对100篇论文进行统一打分。该审稿系统在审稿一致性上已达到人类审稿人判断水平（人类vs人类Spearman相关系数0.41，AI vs人类0.42）。

指标FARS论文ICLR 2026人类投稿ICLR 2026接收论文
平均分5.054.215.39
分数区间3.0 - 6.3——
分数集中区5.2附近（约57次）——

结论：FARS平均分5.05，高于人类投稿平均水平（4.21），但低于接收论文平均分（5.39）。"比下有余，比上未满"——作为全自动系统，已形成相对稳定的"质量带"，偶尔能产出6分以上的高分论文。

6.2 代表性论文深读

成功案例：FA0042 — 双向注意力蒸馏

瞄准文本embedding中双向注意力与KV-cache的矛盾。训练用双向拿质量，推理用因果保效率。通过蒸馏+GG-SM渐进过渡，student模型在LoCoV1长文档检索中NDCG@10达0.284，大幅领先所有baseline（含teacher的0.212）。蚂蚁集团GG-SM发布3天即被FARS接入实验，展现了系统紧跟前沿的敏捷性。

失败案例：FA0121 — Engram稀疏架构修复

盯上DeepSeek的Engram稀疏架构中的"冷热偏置"问题，尝试"反事实门控监督(CGS)"修复。结果基本没救回来——CGS的提升不如多训练几步。论文诚实复盘："Gate和n-gram embedding的训练是一个相互耦合的系统动力学问题，不是简单加监督就能补的。"这种"算法诚实"是当前学术界稀缺的品质。

7.2 FARS的差异化定位

FARS的核心差异化有三：①规模——100篇连续产出是此前无人尝试的规模验证；②透明度——全球首次大规模公开直播部署，全程可观测；③第一性原理设计——跳出学术出版惯例，以"假设+验证"为最小知识单元，鼓励报告失败结果。

9.2 日行迹的独特性

日行迹智能 = 科研型(Research)——在12家Agent平台中，日行迹是唯一不面向企业市场的公司。其他11家都在解决"企业效率"问题（工作流、客服、决策、供应链等），而日行迹解决的是"科研本身"的自动化问题。它不是在优化现有的商业流程，而是在尝试创造新的知识生产方式。这个定位让它脱离了企业级Agent的红海竞争，但也意味着更长的验证周期和更不确定的商业模式。

10.3 发展展望

日行迹智能站在一个历史性拐点上——自动化科研正从"概念验证"走向"工业生产"。智源研究院在2026年趋势预测中指出，AI for Science正从Copilot角色向AI Scientist角色迁移；OpenAI把"全自主AI研究员"写进了2028年路线图；DeepMind宣布将在英国建设首个AI自动化研究实验室。

在这个全球竞赛中，FARS是来自中国的第一个公开实验。228小时100篇论文的验证，首次以工业级规模证明了端到端自动化科研流水线的可行性。团队预测"FARS的引用量将在2026年底超过其所有创造者"——如果成真，这将是AI生成知识超越人类创造者的标志性事件。

一句话总结：日行迹智能是AI Agent赛道中最"纯粹"的科研型公司——以MOSS开发者孙天祥的学术声誉为根基，以FARS四智能体系统为武器，以"AI4AI"为信仰，首次以工业级规模证明了AI自主科研的可行性——但如何将科研自动化转化为可持续商业价值，是这家15人实验室面临的最大生存命题。
