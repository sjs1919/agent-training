01 公司定位与发展历程

一句话定位：全球供应链数字员工领导者，以 AI Agent 驱动企业超自动化。

壹沓科技（ONEAIX Technologies）成立于 2016 年 11 月（2017 年正式运营），总部位于上海市静安区，海外总部设于新加坡。团队约 300 人，核心成员来自华为、阿里、微软等企业。公司是国家级专精特新"小巨人"企业、国家高新技术企业、上海市科技小巨人，拥有 100+ 发明专利与自主知识产权，并牵头参编多项人工智能国家标准。

03 核心技术架构：三引擎闭环

2026 WAIC（7 月 18 日），壹沓科技发布小沓 OS 与小沓 KE 两大核心技术底座，与自研小沓 LLM 协同形成"理解 — 执行 — 学习"完整闭环。CEO 卞晓瑜表示："我们发布的不是两套孤立的技术底座，而是企业级 AI Native 基础设施。"

3.1 小沓 LLM（小沓大模型）— 理解引擎

  • 定位：全球首款供应链垂类大模型，"数字员工的大脑"
  • 技术路径：基于国内外先进大模型基座，融合供应链领域知识、规则、数据，经专业微调后精准解析行业术语与业务指令
  • 核心能力：智能决策、风险控制、流程优化；支持自然语言与语音多模态交互
  • 部署方式：支持私有化部署，满足企业数据安全需求
  • 差异化：不是通用大模型的简单微调，而是将供应链行业规则、SOP、运价逻辑等深度融入模型——这是通用 LLM 无法直接替代的

3.2 小沓 OS — 执行引擎

  • 定位：面向企业复杂流程的 Agent 编排平台
  • 核心能力：
    • 统一编排 Agent、工具与业务流程
    • 任务拆解：将复杂任务分解为可执行子任务
    • 流程控制：多步骤流程的状态管理与异常处理
    • 工具调用：对接企业 ERP/TMS/WMS 等系统
    • 权限约束：角色级权限控制与数据安全
    • 结果校验与反馈回流：执行结果自动验证，异常可追溯
  • 关键意义：让 Agent 在复杂企业场景中"稳定完成端到端任务"，从"能理解"走向"能交付"——这是 Agent 从 Demo 到生产环境的关键一跃

3.3 小沓 KE — 学习引擎

  • 定位：Agent 原生的自进化全域知识引擎
  • 核心能力：
    • 汇聚企业私有数据、行业数据与外部知识
    • 为 Agent 提供精准上下文、长期记忆与持续学习能力
    • 构建面向 Agent 的企业知识底座——可理解、可检索、可调用、可演化
    • 驱动企业知识不断流动、沉淀与进化，形成持续进化的知识闭环
  • 差异化：不只是静态知识库（RAG），而是"Agent 原生"——知识随 Agent 使用自动积累、更新、进化

3.4 CubeFlow — Agent 任务流编排系统

CubeFlow 是连接三引擎的"调度中心"，也是壹沓技术架构的中枢：

  • 多 Agent 协同：按需调度具备数据收集、分析、执行等专长的智能体协同工作
  • 协议支持：基于 MCP（Model Context Protocol）/ A2A（Agent-to-Agent）等协议保障高效交互
  • 复杂流程构建：支持条件分支、循环、异常处理等复杂业务逻辑
  • 可观测性：全程可追溯、异常可预警、结果可保障

3.5 多模态智能解析层

解决供应链场景特有的"信息输入"难题。供应链业务中文本、图像、PDF、邮件等格式混杂，系统通过融合跨模态信息，全面理解内容含义，为上层 Agent 提供高质量的输入支撑。

技术架构小结：壹沓的技术架构设计有一个核心逻辑——解决 Agent 从"能理解"到"能交付"的鸿沟。大多数 Agent 平台停留在 LLM + 工具调用层面，而壹沓通过"OS（编排）+ KE（知识）"补齐了执行稳定性和知识积累两个关键短板，这是其在复杂供应链场景中能真正落地的技术基础。

05 应用场景与量化效果

5.1 供应链物流场景

智能询价报价

痛点：国际物流企业每天对接上千个客户群，人工响应询报价平均需 30 分钟。

解决方案：运小沓询报价 Agent 嵌入企业 IM 系统，客户和销售发条消息就能立刻获得报价，系统还能自动生成客户画像和销售行为分析。

效果：询价响应从 30 分钟缩短到秒级，整体效率提升约 6 倍。"你问我100次，我就可以报100次，生意的逻辑就变了。"—— COO 顾震宇

物流可视化与风险预警

通过物流可视化等工具，AI 追踪货物从工厂到目的港的十几个环节，并提前预警潜在风险（如港口拥堵、罢工等）。这是数字员工赋能业务的关键体现。

单证识别与处理

利用多模态大模型智能解析技术，自动识别提单、舱单、报关单等各类单证，从文本、图像、PDF 等多源异构数据中提取结构化信息，实现单证录入自动化。

5.2 供应链品牌场景

商场结算自动化

案例：某上市女装企业，全国门店超 2000 家，每月需投入近 60 人每天进行 6 小时商场业务结算。

效果：使用壹沓科技平台后，每人每天仅需半小时，效率提升 800%。海澜生活也透露，壹沓自动化平台每年为其节约近 300 万元。

GEO 生成式引擎优化

这是一项前瞻性产品——随着越来越多消费者通过 ChatGPT/文心一言等大模型寻找产品和品牌，GEO 优化确保品牌在 AI 回答中被提及。这是传统 SEO 在 AI 时代的延伸，壹沓在这一领域布局较早。

电商运营自动化

为特步、雅戈尔等品牌提供电商运营自动化：生意参谋日报制作、商品信息统计下载、直通车计划下载、自助分析记录、挽单成功监测等，实现多平台数据一键汇总。

8.2 独特定位

第三方评价（脉脉）指出壹沓的差异化："在供应链以及新零售场景的深度钻研而被置于考虑范围内。特别是针对单据处理、物流数字员工以及大宗商品对账这类的 Know-How，壹沓科技提供了非常成熟的行业套件。"

但也指出局限："对于需要横向打通企业内部多个非供应链系统（如人力资源管理、私域营销以及跨不同框架的定制化后台），壹沓的技术栈与交付手段会体现出一定的场景局限性。"

竞争定位总结：壹沓是"供应链垂直 Agent"路线的代表——深度极深但宽度有限。如果客户的核心痛点集中在供应链履约层面，壹沓是最有竞争力的选择；但如果需要一套统领各部门系统的"中央大脑"，壹沓的横向覆盖能力仍有待验证。

10 护城河、风险与未来展望

10.1 四道护城河

1. 供应链垂直 Know-How 深度沉淀

6 年深耕供应链物流+品牌领域，积累的行业规则、SOP、运价逻辑、单证格式等 Know-How 是通用厂商难以速成的。每一代产品迭代都加深了这一壁垒——从单场景到跨场景到自主规划，行业知识已内化进模型和编排系统。

2. 全球首款供应链垂类大模型

小沓 LLM 不是通用大模型的简单微调，而是将供应链行业知识、规则、数据深度融入。通用 LLM 可以理解"什么是提单"，但无法精准执行运价比对、订舱流程、舱单核验等行业专属逻辑。这一壁垒随行业数据持续积累而加深。

3. "理解—执行—学习"三引擎闭环

小沓 LLM + OS + KE 构成的闭环使壹沓不同于大多数"LLM + 工具调用"的简单 Agent 方案。OS 解决执行稳定性，KE 解决知识积累与进化——这两者在复杂企业场景中是真正的落地门槛。

4. 标杆客户集群与全球化先发

中远海运、中外运、青岛港、DSV 等头部供应链企业客户形成了行业标杆效应。新加坡海外总部 + 5 个月商业化落地速度，使其在国际市场建立了先发优势——国际供应链物流是一个天然全球化的市场，先发优势价值巨大。

10.2 主要风险

1. 赛道宽度局限

深耕供应链是优势也是约束。当客户需要跨部门、跨系统的"中央大脑"时，壹沓的横向覆盖能力不足。若供应链 AI Agent 市场增速放缓，壹沓的天花板也将受限。

2. 大模型厂商的降维打击

大模型能力持续跃升，通用 LLM 可能逐步覆盖供应链垂类 LLM 的部分能力。如果大模型厂商自带 Agent 编排能力进入供应链赛道，壹沓的中间层价值可能被压缩。

3. 从项目制到产品化的转化压力

近 2000 家客户中很多仍需要定制化交付。如何将行业 Know-How 从项目经验转化为标准化产品，是实现规模化扩张的关键挑战。小沓 OS + KE 的发布正是试图解决这一问题。

4. 同赛道竞争加剧

RPA 中国预测 2024 年中国 RPA 市场规模达 77.8 亿元，2022-2026 年复合年增长率 43.4%。来也、影刀、实在智能等厂商都在向 Agent + 垂直行业方向演进，赛道日益拥挤。

10.3 未来展望

CEO 卞晓瑜提出了"AI Native 组织"的演进方向——组织形态正经历三代变迁：

  • 第一代：流程型组织（按 SOP 执行）
  • 第二代：可迭代产品型组织（敏捷开发）
  • 第三代：基于 Agentic 驱动的 AI Native 组织（Agent 自主决策与执行）

壹沓的战略是构建"All-in-One AI"产品，以"白领 + AI Agent"新型办公模式，推动企业从"人主导、AI 辅助"向"Agent 自主决策与执行"跃迁。

核心判断：壹沓科技代表了 AI Agent 落地的"垂直深耕"范式——不做通用平台，而是在供应链这一个垂直赛道上做到极致深度。其护城河不是单一技术，而是"垂类 LLM + 行业 Know-How + 执行闭环 + 标杆客户"的四重复合壁垒。这一路线的天花板取决于供应链 AI 市场的总规模，但在其专注的领域内，壹沓已经建立了难以撼动的领先地位。2026 年小沓 OS + KE 的发布，标志着壹沓正从"供应链 Agent 产品"向"企业级 AI Native 基础设施"升级——如果能成功实现从项目到平台的转化，壹沓有望成为供应链领域的"AI 操作系统"。
