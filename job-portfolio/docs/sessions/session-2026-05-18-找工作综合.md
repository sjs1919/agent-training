## 主题二：项目负责人优化制度（现公司管理成果）

**用户**：要求阅读 `E:\workspace\codebuddy_workspace\projects\big-data-obj\docs\260514-项目负责人优化`

这是用户在**未来工场**设计并推行的**项目负责人权责优化制度**，一套完整的人才梯队培养+项目管理实战体系。

### 核心内容摘要

**制度本质**：在小团队（约16人）中，让有潜力的开发轮流担任"项目负责人"，通过积分奖惩（1分=1天=7工时，与绩效星级直接挂钩：榜首4星1.2倍工资，榜尾2星0.8倍）驱动其从"只写代码"成长为"能带项目"的骨干。

**六大模块**：

1. **权责相匹**：奖励1-3分，惩罚1-3分，项目组成员减半连带责任。新手保护期（前2个项目只记积分不扣分）。需求变更需留痕免责。

2. **需求评审与反讲**：确定主负责人（关键路径所在组），1人周以上项目必须有开发反讲（全员参加，项目成员代讲），输出任务分解+工时评估+联调时间。

3. **开发阶段**：接口提前对齐、每周2次进度会（周三摸底、周五对表）、关注联调时间、测试用例评审。

4. **测试与上线**：统一提测（不允许分别提测）、BUG分级修复（1-2级半天、3-4级24h）、统一发布、double check、发布后观察（1h/1d/1w）。

5. **项目复盘**：数据回顾+亮点总结+问题归因（匿名）+改进措施（聚焦1-2个）+绩效跟踪（积分公开透明）。

6. **八句口诀**：权责相匹、关键路径、反讲必做、每周两回、揪着不放、统一为要、double check、手脚并用。

### 对简历的价值

- 这是具体的"从0到1搭建研发流程"的实战案例
- 包含激励设计、人才培养、流程管控的完整闭环
- 可以展示在小团队（16人）中精细化管理的能力
- 与简历中"建立双周迭代+绩效双循环研发流程"相呼应，可补充更多细节

## 主题三：建立跨设备对话传承机制

**用户**："我需要添加一个规则：每次问答对话，请把对话放到文件夹里；我发现家里对话和公司对话，基本上没有传承"

### 方案讨论

**问题**：Claude Code 对话是本地会话，换电脑就断，家里和公司无法同步。

**我的建议**：把对话核心内容保存为 md 文件到 git 仓库中，通过 git 同步。

**Hook 自动化的局限**：
- Claude Code 的 transcript 是内部 JSONL 格式，路径不固定
- Stop hook 收到的 JSON 输入不包含完整对话内容
- 不同操作系统 transcript 存储位置不同
- 所以纯自动化 hook 不可靠

**最终方案**：
- 每次找工作相关对话结束前，我（Claude）主动用 Write 工具保存
- 保存位置：`shared/resume/docs/sessions/`
- 文件名：`session-YYYY-MM-DD-主题.md`
- 保存粒度：尽可能完整
- **限制条件**：仅找工作/简历/面试相关对话才保存，技术项目开发对话不保存

**规则已写入 CLAUDE.md**：
```
## 对话保存规则（找工作专用）
仅当对话主题涉及找工作、简历、面试、职业规划时，每次对话结束前将完整问答记录保存到 shared/resume/docs/sessions/。
```
