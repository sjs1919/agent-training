## 一、项目背景

**业务场景**：3D打印服务行业，需要统一管理多品牌、多类型的生产设备

**公司规模**：约300人

**挑战**：
- 设备品牌分散：联泰、Formlabs、拓竹、黑格、惠普、华曙、Unicorn、华曙高科等7+品牌
- 设备类型多样：3D打印、尼龙、金属、CNC
- 各品牌API协议不统一，接入成本高
- 设备规模快速增长：不足100台 → 700+台

### 2.2 6个微服务职责

| 服务 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **下载服务** | 从订单系统获取图纸文件 | 订单ID | 原始图纸文件 |
| **解析服务** | 解析图纸格式（STL/STEP等） | 原始图纸 | 标准格式数据 |
| **校验服务** | 校验图纸可打印性、尺寸合规 | 标准格式数据 | 校验报告 |
| **转换服务** | 将图纸转换为设备可识别的切片文件 | 校验通过的文件 | 切片文件（.slc/.cli等） |
| **上机服务** | 通过适配器调用设备API下发打印任务 | 切片文件 + 设备ID | 任务ID |
| **上报服务** | 接收设备状态上报、任务进度反馈 | 设备心跳/状态 | 状态数据入库 |

## 四、架构演进

### 4.1 单集群 → 多集群

**触发原因**：富海产区设备集中度突破联泰软件单集群上限

**演进过程**：
```
单集群架构
  └── 所有设备接入一个集群管理中心
        ↓ 设备数量突破单集群上限
多集群架构
  ├── 集群A（老厂区）
  ├── 集群B（富海产区）
  └── 集群C（备用/扩展）
        ↓ 
负载均衡层：根据设备健康度指标自动分配任务
```

### 4.2 健康度指标体系

用于预判瓶颈、主动触发架构重构：
- 设备在线率
- 任务队列长度
- 平均打印时长
- 故障率趋势
- 集群负载均衡度

## 六、业务流程

```
客户下单
  └──► 图纸解析审核（自动+人工）
         └──► MES排程排产（根据设备状态+交期自动排程）
                └──► 6服务流水线执行
                       └──► 适配器匹配（按设备品牌选择对应适配器）
                              └──► 厂商API上机（设备开始打印）
                                     └──► 设备状态上报（实时回传进度）
                                            └──► 客户可见订单进度
```

---

## 七、技术亮点

1. **适配器模式解耦**：新增品牌只需实现IDeviceAdapter接口，不影响现有流程
2. **流水线容错**：单个服务失败可重试或降级，不影响整体流程
3. **状态上报实时化**：设备状态秒级上报，支持客户实时查看订单进度
4. **自动扩缩容**：多集群架构支持按需扩展，新产区上线周期大幅缩短

### 10.1 MES排程排产（当前实现 + 演进路线）

**当前排程策略**（已上线）：
| 排序维度 | 规则 |
|----------|------|
| 客户重要标签 | VIP客户优先 |
| 生产优先级 | 内部标注的紧急程度 |
| 交期 | 越早越优先 |
| 仓库 | 就近分配 |
| 文件大小 | 大文件优先（大→小） |

**当前特点**：简单排序规则，实现快、逻辑清晰、运营人员易理解

**优化方向（规划中）**：
```
当前：简单排序规则
  ↓
阶段一：引入规则引擎
  - 每个动态进来的订单，通过规则引擎打分
  - 根据打分权重进行排程排产
  - 规则可配置化，支持业务灵活调整
  ↓
阶段二：引入 Agents 智能调度
  - 订单支付后进入 Agent Team 的动态排队
  - 各个环节都动态排队（下载→解析→校验→转换→上机）
  - 确保优先级最高的任务能够在全链路被及时处理
```
