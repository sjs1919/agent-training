# 构建与服务重启规则

## Go 版本约束

- go.mod 锁定 `{{GO_VERSION}}`，**禁止擅自升级**（版本上限通常由生产服务器环境决定，升级须先确认部署环境支持）

## 硬约束

- **注释**：导出标识符 + 包/类型/函数必须有中文注释
- **命名**：包名小写单数，文件小写下划线，枚举 `iota` + 中文注释
- **函数**：≤50 行，参数 ≤4，error 放末位，early return
- **并发**：goroutine 管理生命周期，共享变量加锁，channel 发送方关闭
- **配置**：必有默认值，加载后 `Validate()`，可变配置支持热更新
- **依赖**：锁定版本，标准库 > 已有依赖 > 新引入
- **修改后**：`{{LINT_CMD}}` + `{{BUILD_CMD}}`，保持同目录风格

## 避免重复下载依赖

- Go module / 前端依赖首次下载后缓存持久化，后续构建/测试不要重复拉取（除非新增依赖）
- 批量写代码后一次性编译验证

## 🔴 服务重启三步（强制）

修改代码后需要重启本地服务时，**必须**完整执行，禁止跳过任何一步：

1. **杀掉旧进程**：`taskkill //F //IM {{PROJECT_NAME}}.exe`（确认无残留进程）
2. **编译 + 启动新进程**：`go build -o {{PROJECT_NAME}}.exe ./cmd/ && nohup ./{{PROJECT_NAME}}.exe > logs/server.log 2>&1 &`
3. **确认启动成功**：`sleep 2 && curl -s {{HEALTH_URL}}` 返回存活响应，并检查日志确认启动时间戳

禁止只编译不重启、只杀进程不确认新进程已启动、或使用非标准二进制名（统一 `{{PROJECT_NAME}}.exe`）。
