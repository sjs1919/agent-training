# 测试规范（通用 · 语言无关）

> 本文件定义测试流程与分层原则。语言专属执行细节（框架/命令/Mock 工具）见对应 `stack-*/testing.md`。

## TDD 流程（强制）

```
红（写失败测试）→ 绿（最小实现通过）→ 重构（消除重复/优化结构）
```

- 先写测试，跑一遍确认失败，再写实现代码
- 每完成一个测试用例 → 跑全量确认不破坏已有测试 → 进入下一个
- 修复 bug 必须先写能复现 bug 的测试，确认失败后再修代码

## 三测分层

| 层级 | 比例 | 耗时 | 范围 |
|------|:---:|------|------|
| 单元测试 | ~80% | 毫秒级 | 单函数/单方法，Mock 外部依赖 |
| 集成测试 | ~15% | 秒级 | 多组件协作，真实 DB/Redis/HTTP 直连 |
| E2E 测试 | ~5% | 分钟级 | 完整用户流程，全链路 |

### 单元测试

- 每个导出函数至少一个正向+一个异常用例
- 命名：`TestXxx_场景`（如 `TestLogin_Success`、`TestLogin_InvalidPassword`）
- 结构：Arrange（准备）→ Act（执行）→ Assert（断言），三段清晰
- 禁止依赖环境变量、配置文件、外部服务；对外部依赖一律 Mock（接口+假实现）
- 测试数据用字面量，禁止从文件/DB 动态加载

### 集成测试

- 覆盖：DB 读写事务、缓存穿透/回源、消息队列生产消费、HTTP 中间件链路
- 使用 test fixture（预置数据）+ 每个用例前后自动清理
- 集成测试失败 = 不允许部署

### E2E 测试

- 覆盖核心业务 golden path 2-5 条：注册→登录→核心操作→退出
- 冒烟脚本兜底（`08-test/smoke-test.sh`）：健康检查 + 核心接口 200 OK
- 部署后先跑冒烟，冒烟不通过则回滚

## 测什么 / 不测什么

| 必须测 | 不用测 |
|--------|--------|
| 业务逻辑（计算规则/状态机/权限判断） | 框架样板代码（路由注册/ORM 自动方法） |
| API 契约（响应结构/错误码/字段类型） | 已测依赖的内部实现 |
| 安全路径（未登录拒绝/越权拒绝/输入校验） | 第三方 SDK 内部逻辑 |
| 错误处理路径（DB 宕机/上游超时/参数非法） | 纯 getter/setter、常量定义 |
| 边界条件（空值/零值/最大值/过期/溢出） | 日志格式、注释 |

## 判断矩阵

| 场景 | → | 测法 |
|------|---|------|
| 改了业务逻辑 | → | 旧测试必须 PASS + 新增覆盖新分支 |
| 新增 API 端点 | → | 单测 handler 逻辑 + 集成测 DB 读写 + curl 录一次 E2E |
| 涉及权限/认证 | → | 必测未登录 401 + 越权 403 |
| 涉及金额 | → | 必测正数/零/负数/精度边界 |
| 重构（行为不变） | → | 旧测试全 PASS 即可，不强制加新 |
| 加日志/注释/格式 | → | 不强制新测试 |

## 前端测试

| 类型 | 框架 | 范围 |
|------|------|------|
| 组件逻辑 | Vitest + @vue/test-utils | composable / store / 校验函数 |
| UI 交互 | Vitest + @vue/test-utils | 按钮点击/表单提交/条件渲染 |
| 视觉回归 | Playwright 截图对比 | 核心页面 |
| E2E | Playwright | 关键用户流程（登录→操作→退出） |

组件内纯模板/样式调整跑现有截图对比即可，不写新逻辑测试。

## 质量门禁

- 提交前 `{{TEST_CMD}}` 必须全 PASS，禁止提交失败测试
- 覆盖率 ≥ 80%（核心业务逻辑），CI 中设阈值卡点
- **读输出，不只看「没有红色」** — 确认 PASS/Fail 数量符合预期
- 安全相关路径（认证/授权/输入校验）必须有专门测试，缺失 = 阻塞合并
