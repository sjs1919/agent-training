# {{PROJECT_NAME}} 文档仓库

{{PROJECT_NAME}} 项目所有文档的集中仓库，按开发流程阶段编号组织。与代码仓分离（双仓模式）：spec / plan / todo 全部写在本仓，代码仓只保留编码规范、API 文档、部署指南等与代码同包提交的文档。

> **凭据管理**：所有敏感信息（密码/密钥/Token/API Key）集中存放在 `credentials.local.md`（gitignore，不入库）。首次启用：复制 `credentials.local.md.example` 为 `credentials.local.md` 并填入真实值。文档与代码中只允许出现占位符。

## 目录结构

```
<project>-docs/
├── README.md                 # 本文件
├── credentials.local.md      # 凭据映射（gitignore — 禁止提交）
├── 00-taskregister/          # 待办登记 — 索引 + 单日待办（对话中出现的待办必须登记）
├── 01-requirements/          # 需求 — 原始需求与分析文档
├── 02-specs/                 # 规格 — 功能规格说明书（验收标准所在，门禁②锚点）
├── 03-design/                # 设计 — API 契约、错误码、算法文档
├── 04-plans/                 # 计划 — 实现计划（每任务带自动验收命令，门禁③锚点）
├── 05-tasklist/              # 任务 — 纯状态索引（验收细节以 plan 为唯一源）
├── 06-database/              # 数据库 — 表结构、迁移、关系图
├── 07-process/               # 过程 — 设计决策、实现状态、测试进度
├── 08-test/                  # 测试 — 用例、方案、冒烟脚本（自动挡门禁 2 的机器判定入口）
├── 09-reports/               # 报告 — 评估/审计/测试报告、四门禁证据日志
├── 10-deployment/            # 部署 — 部署指南、部署交接单（门禁④⑤锚点）
├── 11-manuals/               # 手册 — 用户手册、新手上路
└── 12-resources/             # 外部资料 — 供应商记录、第三方文档
```

空目录以 `.gitkeep` 占位（01 / 06 / 07 / 09 / 11 / 12），各目录用途以上表说明为准。

## 命名规范

- **目录名**：英文，按开发阶段编号（`01-requirements`、`02-specs` …）
- **文件名**：保持原名不动（外部/遗留文件允许中文）
- **规格编号**：`F` 系列 = 前端/用户侧功能，`G` 系列 = 后端/基础设施功能
- **日期格式**：文件名中的日期统一 `YYYYMMDD`（如 `20260613`）

## 文档流转规则

- **规格 → 计划**：`02-specs/` 规格确认后（门禁②），实现计划**必须**写入 `04-plans/`，命名格式 `YYYY-MM-DD-<标识>-plan.md`
- **计划 → 任务**：`04-plans/` 计划确认后（门禁③），任务状态索引写入 `05-tasklist/`
- **实现后核对源头（强制）**：编码完成后**必须**回到源头 spec 和 plan，逐条核对验收标准，确认无遗漏。禁止实现到一半就认为完成
- **归档**：禁止物理删除，加 `_archived-` 前缀；覆盖前须用户同意；新建/改名后同步更新索引
